Objective

Create a website that has user authentication with encrypted data and allows each user to have their own unique data. The website must use MVC/repository pattern along with a server to save the data.

Project Difficulties

Had a problem with handling asynchronous data on page load or on real time action. I could not properly save data to display on screen and would disappear after a refresh.

My Solution

I decided to use observables for my data services that I created in ASP.NET Core. I was able to have real-time data along with being able to emit multiple values at any given time.

For the to-do list, I was able to implement the observables to manage its state during the different POST/DELETE/GET usage and make the list change depending on what the user clicked real-time.

Features

  • Login with JWT Authentication
  • Register page with required fields
  • CRUD API Methods
  • Swagger UI for Testing
  • Route Guards to only allow autheticated users to visit certain pages
  • Each User has their own unique data
  • Dynamic to-do list
