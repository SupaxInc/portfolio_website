Objective

Use ASP.NET Boilerplate framework to develop a new feature that dynamically displays any HTML content as a new page on the website. A simpler version of WordPress, Joomla, or other content management systems (CMS).

Project Difficulties

At the beginning of the project, it was very unoptimized since I was manually creating the components for the specific HTML content that I inserted using the Web API methods.

My Solution

I decided to make the pages to be more dynamic and automatically create the pages depending on how many content have been inserted.

To succesfully do this, I made a dynamic URL parameter in my angular routing module then I subscribe to the parameters depending on the id of the HTML content. The pages will then be automatically loaded on start up and the user will be able to choose which content they want to see.

Features

  • Login with User Authentication
  • Multi-tenant functionality
  • CRUD API Methods
  • Swagger UI for Testing
  • Dynamically creates the pages and routing
